# Design: Discoverability & indexing (Google, GitHub, AI search)

**Change:** discoverability-indexing
**Created:** 2026-07-17

## Technical Approach

Purely additive docs-site changes. GitHub Pages already builds `docs/` with Jekyll (`jekyll-theme-cayman`). We layer on the standard indexing artifacts:

- Enable the allowlisted `jekyll-sitemap` plugin and set `url`/`baseurl` so it emits correct absolute URLs.
- Drop static `robots.txt`, `llms.txt`, `llms-full.txt` at the `docs/` root (Jekyll serves front-matter-less files verbatim).
- Tune `index.md` for search intent.
- Add `INDEXING.md` capturing owner-only steps + external-submission drafts.

No plugin runtime code touched; the only non-`docs/` edits are the mandatory version bump + CHANGELOG.

## Architecture

```
docs/
├── _config.yml       # + plugins: [jekyll-sitemap], url, baseurl   (FR1/AC1)
├── robots.txt        # NEW static — allow all + sitemap URL        (FR2/AC2)
├── llms.txt          # NEW static — short llms.txt index           (FR3/AC3)
├── llms-full.txt     # NEW static — expanded LLM-ingest text       (FR4/AC4)
├── index.md          # keyword-tuned title/description             (FR5/AC5)
├── INDEXING.md       # NEW — owner checklist + submission drafts   (FR6/AC6)
├── privacy.md        # unchanged
└── assets/           # unchanged (hero image)
```

`sitemap.xml` is generated by the plugin at build time — not committed.

## Key Decisions

- **D1 — Default Pages domain** (`chan4lk.github.io/specclaw`), so `baseurl: /specclaw` is required and all indexing URLs are pinned to that base. If a custom domain is added later, update `url`/`baseurl`, `robots.txt`, and `INDEXING.md` in one follow-up.
- **D2 — Ship both `llms.txt` and `llms-full.txt`** (per approval): short index for discovery, full text for direct ingestion.
- **D3 — Static `.txt` files, no front matter** so Jekyll serves them raw (avoids HTML-wrapping and templating). Sitemap URL hardcoded absolute in `robots.txt`.
- **D4 — External awesome-list PRs are drafted, not opened** — content lives in `INDEXING.md`; opening PRs against third-party repos is a maintainer decision (out of scope).

## Risks & Mitigations

- **R1 — `baseurl` breaks existing in-site links.** Mitigation: audit `index.md` links; use absolute GitHub URLs or `{{ site.baseurl }}`-prefixed paths. (AC5)
- **R2 — Non-allowlisted plugin fails Pages build.** Mitigation: only `jekyll-sitemap` (on the GitHub Pages allowlist) is added. (NFR3)
- **R3 — `.txt` files accidentally rendered by Jekyll.** Mitigation: keep them front-matter-free; verify raw content. (edge case)
- **R4 — Can't verify live indexing here.** Mitigation: verify = file/structure/YAML checks + link audit; live crawl + Search Console are owner-only and documented in INDEXING.md.
